# Implementation Plan: Circular Dependency Resolution and Skills Modernization

This plan addresses circular dependency detection and resolution, updates the out-of-date scaffolding scripts and builder skills (`agent-package-builder`, `mcp-builder`, `api-client-builder`), and defines a unified layout for modular code organization, tests, and documentation.

## User Review Required

> [!IMPORTANT]
> The scaffolding script (`scaffold_package.py`) will be updated to produce the new modular subfolder structure:
> 1. Split subfolders for API wrappers (`api/`) and MCP servers (`mcp/`) instead of monolithic `api_client.py` and `mcp_server.py` files.
> 2. Structured modern documentation layout including `/docs` (`index.md`, `overview.md`) and a unified `mkdocs.yml`.
> 3. Standardized flat `tests/` folder pattern to support automated test coverage audits.

### Modernizing Scaffolding & Builder Skills

  - Instead of writing `mcp_server.py` and `api_client.py` in the root package folder, it will write a package subfolder structure:

    - `tests/test_api_auth_errors.py` covering error handling and auth guards.
    - `tests/test_mcp_server_coverage.py` ensuring complete endpoint coverage audit checks.

# Troubleshoot Mealie Database Persistence

## Goal
Resolve the initialization failures in the Mealie stack and ensure the `mealie` and `postgres` services are communicating successfully.

## Findings
Through MCP-native diagnostics, we have uncovered the following:
1. The `mealie_postgres` database container successfully started, but it skipped database initialization because the volume (`/media/storage/apps/mealie/pgdata`) already contained data from a previous run.
2. The `mealie_mealie` application container is stuck in a crash loop because it's failing to connect to the database. The specific error is: `FATAL: password authentication failed for user "mealie"`.
3. Because the database was previously initialized (likely with a different password), Postgres is ignoring the `POSTGRES_PASSWORD=mealie_password_main` environment variable provided by Portainer.

> [!WARNING]
> **User Review Required**
> Since the database volume already contains data that is rejecting our current password, we need to decide how to proceed. The most robust way to achieve a "zero-error state" for a fresh initialization is to wipe the old data in the volume so Postgres initializes correctly with the stack's environment variables.
>
> **Are you okay with wiping the existing `mealie` database data to resolve this initialization failure?**

## Proposed Changes

### Option 1: Clean Initialization (Recommended if data is discardable)
We will temporarily modify the `mealie` stack to include a lightweight cleanup container that will wipe the contents of the `mealie-pgdata` volume. After the volume is empty, we will redeploy the original stack, which will force Postgres to run its initialization script and set the correct password (`mealie_password_main`).

### Option 2: Password Reset (If data must be preserved)
If the existing Mealie data is important and must be kept, we will need to inject an initialization script or temporarily set `POSTGRES_HOST_AUTH_METHOD=trust` to bypass authentication and manually reset the `mealie` user's password inside the database. (Note: Since the database is already initialized, Postgres ignores `POSTGRES_HOST_AUTH_METHOD` when passed as an env var, so this would require modifying `pg_hba.conf` directly or executing a command inside the container).

## Verification Plan

### Automated Tests
- Once the action is taken, we will monitor `docker_get_stack_logs` to verify that `mealie_postgres` initializes without errors.
- We will verify that `mealie_mealie` connects to the database successfully and no longer crash loops.
- We will check the Portainer stack status to ensure it remains `Active` with all tasks running smoothly.

## Architecture Decision: finance-mcp

> [!IMPORTANT]
> **All trading functionality becomes tool-driven via a new `finance-mcp` server.**
> This is the correct architecture because:
> 1. Any agent (Antigravity, Claude Code, external) can invoke trading tools
> 2. The MCP server enforces safety guards (paper-first, position limits)
> 3. Exchange backends are abstracted behind the MCP tool interface
> 4. Heavy compute (qlib backtesting, Kronos inference) routes to `data-science-mcp`
> 5. KG knows which MCP server runs on which hardware — routes accordingly

#### KG-Routed Compute

The KG knows where every MCP server runs:
```
finance-mcp (CompanySoftware) --deployed_on--> trading-node-01
data-science-mcp (CompanySoftware) --deployed_on--> gpu-node-01
```

When a workflow needs heavy compute, the KG routes:
```python
# Query KG for compute target
target = graph_query("MATCH (s:CompanySoftware {name: 'data-science-mcp'})"
                     "-[:deployed_on]->(h:InfrastructureHost) "
                     "RETURN h.hostname")
# Route qlib backtest to that node
graph_orchestrate(action="dispatch", task="qlib_backtest",
                  agent_name="data-science-mcp",
                  target_host=target)
```

### Phase 7: Exchange Backend Agent-Package

#### [NEW] `agent-packages/agents/exchange-backends/`

Standalone agent-package with abstracted exchange implementations:

```python
class ExchangeBackend(Protocol):
    """Abstracted exchange interface."""
    name: str
    def submit_order(self, symbol, side, qty, order_type, price) -> ExecutionResult
    def cancel_order(self, order_id) -> bool
    def get_positions(self) -> list[Position]
    def get_account(self) -> AccountInfo
    def stream_quotes(self, symbols) -> AsyncIterator[Quote]

class AlpacaBackend(ExchangeBackend): ...   # FREE paper + live
class BinanceBackend(ExchangeBackend): ...  # Crypto
class IBKRBackend(ExchangeBackend): ...     # Full asset coverage
class FreqtradeBackend(ExchangeBackend): ... # Via freqtrade REST API
class PaperBackend(ExchangeBackend): ...    # Local simulation (DEFAULT)
```

All configurable via `config.json` `trading.exchanges` section.

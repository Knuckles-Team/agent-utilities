# Increasing Pytest Code Coverage to North Star Goal

Increase the overall pytest code coverage of `vector-mcp` from **32%** towards the north star goal of **100%**.

We will achieve this by implementing robust, offline-friendly unit tests that mock the external databases and APIs, ensuring all dynamic-import retrievers, the FastMCP server, the Agent CLI server, the authentication layer, and the individual database adapter wrappers are tested programmatically.

---

## User Review Required

> [!IMPORTANT]
> **Mocking vs. Real Database Containers:**
> Our integration tests under `tests/test_databases.py` and `tests/test_all_backends.py` rely on running Docker containers (Postgres, MongoDB, Couchbase, Qdrant) and are skipped if they are unavailable.
> To hit our high-coverage target, we will introduce **mock-based unit tests** that mock database adapters, client connections, and embeddings. This allows us to cover 100% of the internal logic (e.g., SQL generation, similarity functions, JSON parsing, configuration checks) offline without relying on active local database containers.

## Proposed Changes

We will group our changes into a series of new unit test files targeting the components with the lowest coverage.

---

### Mocks & Database Adapters

#### [NEW] [test_auth.py](file:///home/apps/workspace/agent-packages/agents/vector-mcp/tests/test_auth.py)
- **Goal:** Exercise `vector_mcp/auth.py`. Since `from vector_mcp.vector_api import Api` points to a missing file/placeholder module, we will mock `vector_mcp.vector_api` during import inside the test to assert that `get_client` reads environment variables (`LLM_BASE_URL`, `LLM_TOKEN`, `LLM_SSL_VERIFY`) and returns an `Api` client successfully.

#### [NEW] [test_vectordbs_mock.py](file:///home/apps/workspace/agent-packages/agents/vector-mcp/tests/test_vectordbs_mock.py)
- **Goal:** Exercise the core database adapter wrappers (`vector_mcp/vectordb/*`) under offline conditions. We will mock:
  - `psycopg` connection and cursor for `PostgreSQL`.
  - `pymongo.MongoClient` and collection operations for `MongoDBAtlasVectorDB`.
  - Couchbase SDK/cluster client and Couchbase REST fallback methods for `CouchbaseVectorDB`.
  - `qdrant_client.QdrantClient` for `QdrantVectorDB`.
- **Coverage Impact:** Tests edge cases, cosine similarity calculations, query formatting, and error handling, lifting `vectordb/*` coverage closer to 100%.

---

### Retrievers & Protocol Implementations

#### [NEW] [test_retrievers.py](file:///home/apps/workspace/agent-packages/agents/vector-mcp/tests/test_retrievers.py)
- **Goal:** Exercise retriever implementations under `vector_mcp/retriever/*` and `llamaindex_retriever.py` (which currently has 0% coverage).
- **Details:**
  - Mock `BasePydanticVectorStore` and test the generic `LlamaIndexRetriever`.
  - Test concrete retrievers (`ChromaDBRetriever`, `PGVectorRetriever`, `MongoDBRetriever`, `QdrantRetriever`, `CouchbaseRetriever`).
  - Cover core protocol methods: `initialize_collection`, `connect_database`, `add_documents`, `query`, and `bm25_query`.
  - Handle inputs like directory loader paths, document list inputs, and raw string contents.

---

### Servers & Command Line Interface

#### [NEW] [test_mcp_server.py](file:///home/apps/workspace/agent-packages/agents/vector-mcp/tests/test_mcp_server.py)
- **Goal:** Exercise the FastMCP server, tool registration, custom routes, and command line execution in `vector_mcp/mcp_server.py`.
- **Details:**
  - Mock the FastMCP context and the `get_client` dependency.
  - Assert that calling `get_mcp_instance()` returns a valid `FastMCP` app with the expected custom routes (like `/health`) and registered tools (`vector_collection_management` and `vector_search`).
  - Test tool execution using the mocked client dependencies.
  - Test the `mcp_server` main CLI entry point under different options (e.g. `stdio`, `sse`).

#### [NEW] [test_agent_server.py](file:///home/apps/workspace/agent-packages/agents/vector-mcp/tests/test_agent_server.py)
- **Goal:** Exercise the Agent CLI parser and server execution in `vector_mcp/agent_server.py`.
- **Details:**
  - Mock `agent_utilities` CLI parser and server creation functions.
  - Use `patch.object(sys, "argv")` to pass CLI args.
  - Assert that all environment values and defaults (identity, name, description) are resolved correctly and passed to `create_agent_server`.

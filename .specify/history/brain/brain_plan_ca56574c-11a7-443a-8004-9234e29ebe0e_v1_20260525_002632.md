# Native Progress Tracking in Phased Bumps & Pushes

We are enhancing the `repository-manager` MCP server and codebase to natively support granular, real-time progress reporting for phased version bumping (`maintain`) and phased git pushing (`phased_push`). This allows tracking the status, counts, and individual repository progress during long-running repository-manager operations, exactly like `validation`.

## Proposed Changes

### 1. `repository_manager/repository_manager.py`

#### [MODIFY] [repository_manager.py](file:///home/apps/workspace/agent-packages/agents/repository-manager/repository_manager/repository_manager.py)
- Update `phased_bumpversion` (also aliased to `maintain_projects`) to accept an optional `progress: dict | None = None` parameter.
- Update `phased_push` to accept an optional `progress: dict | None = None` parameter.
- Implement pre-expansion of phases and repositories at the beginning of these methods to determine the exact total count of operations.
- Initialize the shared `progress` dictionary with standard structure:
  - `current_phase`: string representing the currently active phase
  - `progress`: integer (0-100) representing total percentage of processed projects
  - `phases`: detailed dictionary mapping phase names to their stats (`status`, `total`, `processed`, `completed`, `success`, `failed`, and a `details` mapping of repository names to their status string).
- Update the progress stats in real time as each repository is processed. For concurrent push tasks, update the stats dynamically in the thread pool loop using `concurrent.futures.as_completed`.

### 2. `repository_manager/mcp_server.py`

#### [MODIFY] [mcp_server.py](file:///home/apps/workspace/agent-packages/agents/repository-manager/repository_manager/mcp_server.py)
- Create a shared `progress` dict in `rm_git` for `phased_push` action.
- Create a shared `progress` dict in `rm_projects` for `maintain` action.
- Pass `progress=progress` and `_extra_job_data={"progress_detail": progress}` to `_submit_job`.
- Update the tool definitions to describe this capability.

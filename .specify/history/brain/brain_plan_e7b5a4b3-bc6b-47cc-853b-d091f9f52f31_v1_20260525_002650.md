# Code Enhancement Audit for Agent Packages

This plan outlines the execution of the `code-enhancer` tool across all 37 sub-projects under `agent-packages/agents/*`. The audit will run multiple specialized domain checks in parallel across all projects, generating structured SDD specs, checklists, tasks, and code quality reports.

---

## Approved Parameters

> [!NOTE]
> **Audit Scale & Resource Consumption**
> - **Concurrency limit**: `c = 4` parallel project workers (approved to manage system resource usage).
> - **Output location**: `/home/apps/workspace/reports` (approved location for the unified comparison table, summaries, and results).
> - **Special Focus**: **`emerald-exchange`** was recently created. We will ensure it is fully analyzed and compliant with standard packaging, prompt externalization, test coverage, and documentation schemas.

> [!IMPORTANT]
> **Automated SDD Specification Handoff**
> The `code-enhancer` tool formats its findings as SDD functional requirements, user stories, and tasks, writing them directly to each project's own `.specify/` directory as:
> - `.specify/specs/code-enhancement-[date]/spec.md` (User stories, functional requirements, success criteria)
> - `.specify/specs/code-enhancement-[date]/spec.json`
> - `.specify/specs/code-enhancement-[date]/tasks.md` (Actionable checklist items sorted by priority)
> - `.specify/specs/code-enhancement-[date]/tasks.json`
>
> This treats each project's `.specify/` folder as a local Single Source of Truth, enabling downstream agents or developer tools to directly resolve them.

## Verification Plan

### Automated Execution & Validation
We will run the parallel runner and monitor progress. Once finished, we will verify:
1. `summary.json` is generated correctly in the reports folder.
2. Per-project `.specify/` folders contain their respective spec and task lists.

As mandated by the standard `code-enhancer` design, after generating or modifying files in the `.specify/` folders, we will invoke the `kg_ingest` tool to sync these changes directly into the Knowledge Graph to ensure complete ecosystem visibility.

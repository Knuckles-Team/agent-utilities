# Ansible Tower MCP Codebase Enhancement & SOLID Architecture Plan

This document outlines the detailed architectural, testing, and documentation enhancement plan for `ansible-tower-mcp` to address all findings from our `code-enhancer` audit and improve the project quality score to elite standards.

---

## User Review Required

We propose structural changes to the codebase and test suite that will greatly improve modularity, maintainability, and clean separation of concerns:

1. **Extracting Tools from `mcp_server.py`**:
   - Currently, `mcp_server.py` is a 707-line "god module" holding both the runner/CLI logic and the tool definitions for all 15 Ansible Tower capabilities.
   - We propose extracting the tool definitions to a new `ansible_tower_mcp/tools.py` file, leaving `mcp_server.py` as a highly cohesive, lightweight bootstrapping runner of less than 100 lines.
2. **Organizing and Restructuring the Test Suite**:
   - The test directory is flat and contains a single massive 637-line file (`tests/test_coverage_edge_cases.py`) along with redundant brute-force test files.
   - We propose restructuring the test directory into `tests/unit/` and `tests/integration/` subdirectories, creating a shared `tests/conftest.py` for standard fixtures, and splitting the monolithic edge case file into focused, single-responsibility files.

> [!IMPORTANT]
> All changes are fully backward-compatible and will not affect runtime tool behavior or command-line execution interfaces.
